## Part 1: Before Your Presentation (5 minutes prep)

### Checklist

Before presenting, make sure you can:
- [ ] Run your project without errors
- [ ] Explain what it does in one sentence
- [ ] Show the most interesting feature
- [ ] Discuss one challenge you overcame
- [ ] Share one thing you'd do differently

### Presentation Structure (5-7 minutes)

1. **Introduction (30 sec)**: What is your project? What problem does it solve?
2. **Demo (2-3 min)**: Show it working! Walk through the main features.
3. **Code highlight (1-2 min)**: Show one piece of code you're proud of.
4. **Challenges (1 min)**: What was hardest? How did you solve it?
5. **Reflection (30 sec)**: What would you do differently? What did you learn?

### Tips for a Great Presentation

**Do:**
- Practice beforehand (even just once!)
- Speak slowly and clearly
- Show enthusiasm for what you built
- Be honest about challenges

**Don't:**
- Apologize excessively ("This is bad, but...")
- Read code line by line
- Rush through the demo
- Ignore questions

## Instructor Notes

### Timing Tips

- Keep presentations to time (use a visible timer)
- Allow 1-2 questions per presentation if time permits
- Have a backup plan for technical difficulties

### Creating Safe Space

- Model positive feedback yourself
- Celebrate effort, not just results
- Acknowledge that presenting is hard

### Handling Issues

- Student's project doesn't work? Let them explain what it was supposed to do
- Student is too nervous? Offer to present alongside them
- Running short on time? Skip Q&A, do written feedback only
