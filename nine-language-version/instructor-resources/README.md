- **10 min**: Discussion: compilation vs interpretation

### Course Philosophy

**Why 9 Languages?**
- Not to make students "experts" in all 9
- To develop **paradigm fluency** and **transferable concepts**
- To show that programming principles transcend syntax
- To prepare students for learning ANY language in the future

**Teaching Approach:**
- **Compare constantly**: "How would this work in Python vs Haskell?"
- **Focus on concepts**: Variables are the same idea across languages
- **Embrace confusion**: It's okay to not master every language
- **Celebrate differences**: Languages have different philosophies for good reasons

### Time Commitment

**Per Lesson (estimate):**
- Student reading: 1-2 hours
- Exercises: 2-4 hours
- Class time: 2-3 hours
- Total: 5-9 hours per lesson

**Full Course:**
- 5 lessons completed = 25-45 hours (about 1-2 weeks full-time or 3-5 weeks part-time)
- 10 lessons (if completing all) = 50-90 hours (half a quarter)

### Assessment Philosophy

**Formative Assessment:**
- Exercises are for learning, not grading
- Encourage experimentation and failure
- Value understanding over completion

**Summative Assessment:**
- Projects (see practice-projects/)
- Final exam with short programs in multiple languages
- Reflection essay on paradigm differences

**Grading Focus:**
- Can student explain the WHY, not just the HOW?
- Can student transfer concepts between languages?
- Can student choose appropriate language for task?

### Common Student Questions

**"Do I need to master all 9 languages?"**
No! The goal is conceptual understanding. You should:
- Be comfortable in 2-3 languages
- Understand the paradigms all languages represent
- Be able to read and modify code in all 9

**"Which language should I focus on?"**
Depends on goals:
- **Industry job**: Python, JavaScript, Java
- **Systems programming**: C, Rust
- **Functional programming**: Haskell, Racket
- **Logic programming**: Prolog
- **My advice**: Pick one from each paradigm

**"This is too hard/too easy"**
- **Too hard**: Focus on fewer languages, use Python/JavaScript first
- **Too easy**: Dive deeper into one paradigm, try the challenge projects

### Diversity and Inclusion

**Make the course accessible:**
- Not everyone has the same computing background
- Some students may have only seen Python or Java
- Some may have never programmed before
- Language barriers: code syntax is language-agnostic

**Create safe learning environment:**
- No "stupid questions" about syntax
- Celebrate different learning speeds
- Pair programming helps (mix experience levels)
- Office hours for individual help

### Pacing Recommendations

**Option 1: Intensive (1-2 weeks)**
- 1 lesson per day
- 2-3 hours class + 4-6 hours independent work
- Good for: Boot camps, intensive workshops

**Option 2: Standard (5-10 weeks)**
- 1 lesson per week
- 2-3 hours class per week
- 3-6 hours homework per week
- Good for: University courses, evening courses

**Option 3: Self-paced**
- Students work at own speed
- Weekly check-ins or office hours
- Good for: Online courses, independent study

### Live Coding Tips

**Best practices:**
1. **Start simple**: Begin with working code, then modify
2. **Make mistakes**: Show debugging in action
3. **Think aloud**: Explain your reasoning as you type
4. **Compare languages**: Write same program in 2 languages side-by-side
5. **Use student ideas**: "How would you solve this?"

**Demo ideas:**
- Lesson 1: "Hello, World!" in 3 languages (different paradigms)
- Lesson 3: FizzBuzz comparison (imperative vs functional)
- Lesson 4: Build a closure in Python vs Haskell
- Lesson 5: Demonstrate aliasing bug and immutable solution

### Office Hours Strategy

**Common topics:**
- Installation issues (especially Haskell, Prolog)
- Syntax confusion ("Why does Haskell use `:` for cons?")
- Conceptual gaps (closures, immutability)
- Exercise help (guide, don't solve)

**Effective approaches:**
- Ask students to explain their thinking
- Use whiteboard for concepts, not just code
- Compare to language they know
- Suggest simpler problem first

## Quick Reference

### Lesson Difficulty Ratings

- **Lesson 1** (Easy): Hello World, compilation vs interpretation
- **Lesson 2** (Easy-Medium): Variables, types, mutability
- **Lesson 3** (Medium): Control flow, pattern matching
- **Lesson 4** (Medium-Hard): Functions, closures, purity
- **Lesson 5** (Medium-Hard): Data structures, mutability vs immutability

### Language Difficulty Ratings (for students)

**Easiest to start:**
1. Python - familiar syntax, forgiving
2. JavaScript - similar to Python
3. Ruby - intuitive, friendly

**Moderate:**
4. Java - verbose but clear
5. C - simple but unforgiving
6. Rust - clear but strict

**Most challenging:**
7. Haskell - different paradigm
8. Racket - parentheses, recursion
9. Prolog - logic programming is unusual

**Note**: "Challenging" doesn't mean "worse"! Just different thinking.

### Emergency Teaching Mode

**If you're short on time:**
1. Focus on Lessons 1-3 (fundamentals)
2. Pick 3-4 languages to demonstrate deeply
3. Have students try others independently
4. Use pre-built test scripts to verify work
5. Skip some exercises, focus on concepts
